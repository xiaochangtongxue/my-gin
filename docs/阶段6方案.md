### 6.1 限流器（Rate Limiter）

**目标**：实现 IP/用户级别的请求限流，防止服务过载。

#### 技术选型
- **golang.org/x/time/rate** - 令牌桶算法
- **Redis** - 分布式限流支持

#### 架构设计

```

├── limiter.go       # 限流器接口定义
├── token_bucket.go  # 令牌桶实现

internal/middleware/
└── ratelimit.go     # 限流中间件

└── struct.go        # 添加 RateLimitConfig（修改）
```

#### 限流器接口

```go
type Limiter interface {
    Allow(ctx context.Context, key string) bool
    AllowN(ctx context.Context, key string, n int) bool
}
```

#### 配置项

```yaml
middleware:
  ratelimit:
    enable: true
    # 全局限流
    global:
      rate: 100      # 每秒请求数
```

#### UID 生成算法（Feistel Cipher）

```go
// pkg/utils/uid.go
import "math/bits"

const uidPrefix = 10000000000000  // 前缀，确保14位

// feistelEncode 可逆混淆
func feistelEncode(id uint64) uint64 {
    const rounds = 3
    l := id >> 22
    r := id & 0x3FFFFF
    for i := 0; i < rounds; i++ {
        next := l ^ roundFunction(r, i)
        l, r = r, next
    }
    return (l << 22) | r
}

func roundFunction(val uint64, round int) uint64 {
    key := uint64(374601907 + round*31)
    return bits.RotateLeft64(val*key+key, 11) & 0x3FFFFF
}

// EncodeUID 数据库自增ID混淆为14位UID
func EncodeUID(id uint64) uint64 {
    return uidPrefix + feistelEncode(id)
}
```

**生成流程**：
```
1. INSERT users → 获取自增 id
2. uid = EncodeUID(id)
3. UPDATE users SET uid = ? WHERE id = ?
```

#### 登录流程

```
┌─────────────────────────────────────────────────────────────────┐
│  POST /api/v1/login                                              │
│  { mobile, password, captcha_id?, captcha_code? }                │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 根据 mobile 查用户（不验证密码）                               │
│     → 不存在 → 返回 "用户不存在"                                  │
│     → 存在 → 获取 uid                                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 检查账号是否被锁定                                             │
│     Redis: login:locked:{uid}                                    │
│     → 已锁定 → 返回 "账号已锁定，请XX分钟后再试"                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 检查是否需要验证码                                             │
│     Redis: login:need_captcha:{ip}:{uid}                         │
│     → 需要 → 验证 captcha_id + code                              │
│              → 错误 → 返回 "验证码错误"                           │
│              → 正确 → 删除 captcha_id，继续                       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 验证密码（bcrypt）                                            │
│     → 正确 → 删除失败计数、删除need_captcha标记、返回Token        │
│     → 错误 → 失败次数+1、设置need_captcha标记、检查是否锁定       │
└─────────────────────────────────────────────────────────────────┘
```

#### Redis 数据结构

| Key | Value | TTL | 用途 |
|-----|-------|-----|------|
| `captcha:{captcha_id}` | 验证码答案 | 5分钟 | 验证码存储 |
| `login:need_captcha:{ip}:{uid}` | "1" | 15分钟 | 需要验证码标记 |
| `login:fail:{ip}:{uid}` | 失败次数(int) | 15分钟 | 失败次数计数 |
| `login:locked:{uid}` | "1" | 30分钟 | 账号锁定 |
| `login:blacklist:{ip}` | "1" | 1小时 | IP黑名单 |

#### 防护策略

| 策略 | 触发条件 | 锁定时间 |
|------|----------|----------|
| IP 限制 | 同一IP 5次/15分钟 | 锁定 30分钟 |
| 账号限制 | 同一账号 3次/15分钟 | 锁定 30分钟 |
| IP 黑名单 | 累计 10次失败 | 锁定 1小时 |

#### 权限模型（model.conf）

```
[request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

[role_definition]
g = _, _

[policy_effect]
e = some(where (p.eft == allow))

[matchers]
m = g(r.sub, p.sub) && KeyMatch2(r.obj, p.obj) && r.act == p.act
```

   - 实现限流中间件
   - 添加配置项

2. **防暴力破解**（优先级高）
   - 创建 Security 服务
   - 实现失败记录和锁定逻辑
   - 添加配置项和错误码
   - 集成到登录流程

3. **RBAC 权限**（优先级中，依赖较多）
   - 集成 Casbin 和 GORM 适配器
   - 创建数据库迁移
   - 实现 RBAC 中间件

   - 实现权限管理服务
